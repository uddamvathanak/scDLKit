# Milestone 6: paper assets and manuscript

Status: planned

## Objective

Turn the benchmark program into a `Bioinformatics`-ready evidence package and
manuscript outline.

## Why this matters for the paper

The software alone is not the publication. The publication requires a claims
audit, a figure and table inventory, a frozen evidence set, and a manuscript
shape that matches the target journal rather than a generic project report.

## Current state

- the planning target is now a `Bioinformatics` Original Paper
- M1 annotation is the first validated paper pillar, but it is not the full
  paper on its own
- some report and figure artifacts already exist for annotation
- there is no complete paper inventory yet
- the manuscript has not yet been budgeted against the journal's 7-page limit
- there is no structured abstract, cover-letter argument, or claims-versus-
  evidence map for the target journal yet

## Exit artifacts

- target journal and article-type lock
- figure inventory mapped to the journal page budget
- table inventory mapped to the journal page budget
- frozen benchmark artifact set
- manuscript outline in `Bioinformatics` section order
- structured abstract in `Bioinformatics` format
- claims versus evidence audit
- submission package checklist

## Risks / blockers

- the paper can become a software description without strong evidence if figures
  and tables are not planned early
- the manuscript can miss the journal fit if it reads like a roadmap instead of
  a finished computational biology contribution
- the current full-toolkit target can overclaim if M2-M5 evidence is not landed
  before writing final claims

## Dependencies

- all four task pillars
- cross-model PEFT expansion
- `planning/research/bioinformatics_original_paper_target.md`

## Acceptance criteria

- the paper can be written from a defined evidence inventory rather than from
  scattered repo artifacts
- the manuscript package is explicitly shaped to a `Bioinformatics` Original
  Paper rather than a generic software note
- the section order, structured abstract headings, figure budget, and
  supplementary-material split are all defined before drafting the paper
- the code, data, and artifact availability package expected at submission is
  explicit
- there is no mismatch between the claims in the manuscript plan and the
  milestone evidence actually present in the repo
